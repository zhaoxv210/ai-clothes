# AI 电子衣橱 · 乔布斯式重新设计

## 核心理念

遵守乔布斯式产品思维：极简主义、直觉交互、像素级质感、聚焦核心场景。

## 架构：两屏结构

```
今日穿搭（主页）← 底栏 → 衣橱管理
```

删除侧边栏、日历页、统计页、设置页。app 打开即用，没有学习曲线。

## 页面设计

### 1. 今日穿搭（主页）

- **打开 app 即看到**「今天穿什么？」推荐卡片
- **顶部天气：** 一行显示温度 + 天气（免费 API），无加载状态
- **推荐卡片：** 大卡片居中，缩略图堆叠排列，下方文字显示品类 + 名称
- **[换一套]：** 重新随机推荐
- **[就穿这个]：** 标记今日穿搭，写入本地记录
- **空白状态：** 大 "+" 按钮引导添加第一件衣服，"还没有衣服，先添加一些吧"

### 2. 衣橱管理

#### 浏览模式
- **大图网格：** 3 列正方形卡片，缩略图居中
- **筛选：** 顶部展开毛玻璃面板，按品类筛选
- **搜索：** 筛选面板展开后出现搜索框
- **点击卡片：** 全屏展示详情
- **[+添加]：** 右下角圆形按钮

#### 添加流程
拍照/选择图片 → 抠图处理（后台不阻塞） → 填写表单 → 自动保存返回浏览

表单字段：名称、品类、季节、品牌、颜色（共 5 个字段）

#### 详情/编辑/删除
- 全屏大图 + 底部信息（名称、品类、季节、品牌）
- **[编辑]：** 推入编辑页
- **[删除]：** 直接删除，无确认弹窗。显示 toast "已删除" + 1 秒内可撤销

## 推荐逻辑

1. 排除今日已穿
2. 匹配当前季节（6 月不推羽绒服），季节判定：春 3-5、夏 6-8、秋 9-11、冬 12-2
3. 组合：1 上装 + 1 下装 + 1 鞋。缺品类则跳过该品类，至少出 1 件
4. 随机不重复：换一套时从没推过的选，循环后重置
5. 如果所有衣服季节都不匹配，忽略季节条件，出任意组合

**不实现：** AI 风格分析、颜色匹配、穿搭评分、智能学习

## 交互原则

- **无保存按钮：** IndexedDB 自动持久化
- **点击为主，** 无拖拽操作
- **删除无二次确认，** 用 toast 撤销替代
- **所有转场平滑，** 暗黑色系毛玻璃质感

## 技术栈（不变）

- Vite + React + TypeScript
- Tauri v2（桌面原生）
- Dexie (IndexedDB)
- Zustand（轻量状态）
- wouter（hash 路由，尽管只有 2 屏）
- Tailwind CSS v4
- `@xixixiao/rembg`（抠图，ONNX runtime）
